# Project Summary

DCraft Fusion is the control plane OS for modern data platforms. It coordinates existing data tools, metadata systems, orchestration engines, transformation engines, BI tools, and governance systems through one governed operating layer.

The project principle is: Fusion is the brain; existing engines are the muscles.

# Project Module Description

- **Control Plane Kernel**: Tenants, projects, connections, workflows, runs, policy, audit, adapters, and AI assist.
- **Provider/Adapter System**: Typed provider contracts, capability discovery, health, config validation, translation, execution, status, lineage extraction, and optional event watching.
- **Metadata and Lineage**: Discovery, resource models, lineage graph, drift detection, and reconciliation.
- **AI Operator**: Recommendation and draft generation with approval-gated execution.
- **Deployment Models**: SaaS, customer VPC/BYOC, on-prem, and air-gapped support.
- **Future Platform Layers**: CDC, transformation, semantic layer, BI/reporting, governance, observability, cost, and automation.

# Technology Stack Direction

- Backend: Java/Spring Boot or Go for core services; Python/FastAPI where useful for AI, metadata, connectors, or workers.
- Frontend: React and TypeScript.
- Internal contracts: gRPC and typed resource contracts.
- Frontend-facing APIs: REST and/or GraphQL.
- Metadata store: PostgreSQL.
- Cache/state: Redis.
- Event bus: Kafka/Redpanda or NATS depending on implementation phase.
- Infrastructure: Docker, Kubernetes, Helm, Terraform, and local-dev tooling.

# Usage

This repository is currently scaffolded for implementation. Start with the control plane kernel before building CDC, BI, or advanced AI automation.
